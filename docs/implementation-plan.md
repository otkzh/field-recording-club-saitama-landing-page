# フィールドレコーディングクラブさいたま

## サンプルWebページ実装計画

## 1. 目的

「フィールドレコーディングクラブさいたま」の活動紹介、今年度事業の告知、サウンドスケープ感のある音地図体験を、シンプルな1枚ページとして実装する。

初期版では、完成版アーカイブではなく、以下を確認できるサンプルページを作る。

- 団体・プロジェクトの雰囲気が伝わる
- MapLibreベースの音地図が表示される
- 地図上の地点をクリックすると音源情報が出る
- SoundCloud埋め込み音源を再生できる
- イベント参加・SNS導線がある
- 後から実音源・写真・イベント情報を差し替えやすい

## 2. ページの基本方針

### コンセプト

耳をひらくと、さいたまは別の風景になる。

### ページ種別

1ページ完結型のランディングページ。

1. 見た目と雰囲気

## 5. SoundCloud音源の扱い

### 5.1 初期サンプル

初期実装では、SoundCloud上で埋め込み可能なフィールドレコーディング系・環境音系の公開音源を仮置きする。

ただし、第三者の音源をページの正式コンテンツとして見せると権利・文脈の問題があるため、公開時には以下のいずれかに差し替える。

- FRCSが録音した音源
- ワークショップ参加者の許諾済み音源
- アーティスト本人から許諾を得た音源
- Creative Commons等の利用条件が明確な音源

### 5.2 実装方針

SoundCloud URLをGeoJSONのプロパティとして持たせる。

```json
{
  "type": "Feature",
  "properties": {
    "id": "omiya-station-east",
    "title": "大宮駅東口の音",
    "place": "大宮駅東口",
    "description": "駅前の人の流れ、バス、信号音、建物に反射する環境音。",
    "note": "都市の音が層になって聞こえる地点。",
    "soundcloudUrl": "SOUNDCLOUD_TRACK_URL",
    "photo": "assets/img/omiya-station.jpg",
    "tags": ["urban", "station", "traffic"]
  },
  "geometry": {
    "type": "Point",
    "coordinates": [139.624, 35.906]
  }
}
```

### 5.3 プレイヤー表示

初期版では、クリックした地点のSoundCloud URLをもとにiframeを差し替える。

- マーカークリック
- 情報パネルを更新
- SoundCloud iframeを表示
- 自動再生はしない

将来的に再生制御を細かくしたい場合は、SoundCloud Widget APIを使う。

## 6. MapLibre実装方針

### 6.1 地図の中心

初期表示は大宮から見沼周辺を中心にする。

```text
中心座標：大宮駅〜氷川参道周辺
ズーム：12〜13程度
```

### 6.2 地図デザイン

通常の道路地図そのままではなく、音風景に合うように落ち着いた配色を使う。

- 背景は薄め
- 道路は細く
- 水辺や緑地が見える
- マーカーは録音ボタンを思わせる小さな円
- 選択中の地点だけ強調

### 6.3 マーカー表示

初期版ではGeoJSON sourceとcircle layerで表示する。

- 通常マーカー：小さな円
- hover時：少し拡大
- 選択中：色またはサイズを変更
- popupではなく、右側または下部の情報パネルを更新する

## 12. 実装時の注意点

### 12.1 音源の権利

サンプル実装では外部音源を仮置きできるが、正式公開では以下を確認する。

- 埋め込み許可
- 利用ライセンス
- クレジット表記
- 録音者の許諾
- 録音対象に人物の会話などが含まれていないか

### 12.2 自動再生しない

音を扱うページだが、自動再生は避ける。

- ユーザー体験を損なう
- モバイルブラウザで制限される
- アクセシビリティ上望ましくない

### 12.3 音だけに依存しない

各音源には必ず説明文を付ける。

- 何の音か
- どこで録音されたか
- どのような風景か
- どのように聴いてほしいか
